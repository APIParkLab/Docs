# 快速入门

欢迎使用`APIPark`！为了帮助您快速上手并充分利用平台的功能，我们为您提供了详细的快速入门指南。该指南分为三个主要部分：前置工作、服务提供方、服务订阅方。通过这些步骤，您将了解如何准备环境、发布API服务以及订阅和使用API服务。

#### 前置工作

在开始使用API开放平台之前，您需要完成一些前置工作。这部分将指导您完成基础设置，确保您拥有顺利的操作体验。

1. **新建分区**：
   * 创建不同的分区来管理不同环境的网关集群，如生产环境、测试环境和开发环境，或者按地域划分，如华北、华东、华南等。
   * 详细说明如何创建分区并配置其参数。
2. **配置监控**：
   * 设置监控功能以收集和展示访问分区的请求访问数据，使用可视化图表来展示数据，帮助您实时监控API的运行状况。
   * 提供监控配置步骤和图表说明。
3. **新建组织**：
   * 建立和管理您的组织架构，可以包括内部公司或合作伙伴。
   * 说明如何创建组织、添加成员并分配角色和权限。
4. **新建团队**：
   * 在组织内创建多个团队，每个团队有自己的系统和权限配置，确保任务和资源的明确划分。
   * 详细说明如何创建团队并配置其系统和权限。

#### 服务提供方

作为服务提供方，您将发布和管理API服务，供其他用户或系统调用。以下是服务提供方需要完成的主要步骤：

1. **新建服务**：
   * 创建一个新的API服务，定义服务名称、描述和基本配置。
   * 说明如何创建服务和配置基本信息。
2. **配置上游**：
   * 配置服务的上游地址，确保API请求能够正确路由到后端服务。
   * 提供上游配置的详细步骤和注意事项。
3. **新建API**：
   * 使用平台提供的工具定义您的API，包括接口、参数、返回值等。
   * 提供API定义的详细指南和示例代码。
4. **配置服务并绑定API**：
   * 将定义好的API绑定到具体的服务上，并进行相关配置，如访问控制、限流策略等。
   * 说明如何绑定API和配置服务策略。
5. **发布服务版本**：
   * 将配置完成的服务版本发布到平台上，使其可以被其他用户或系统调用。
   * 介绍如何发布服务版本和管理不同版本的API。
6. **审批订阅**：
   * 审核和审批其他用户或系统的API订阅申请，确保只有授权用户能够访问API服务。
   * 说明审批订阅的流程和注意事项。

#### 服务订阅方

作为服务订阅方，您可以订阅和使用API服务，实现与其他系统的集成。以下是服务订阅方需要完成的主要步骤：

1. **新建应用**：
   * 创建一个新的应用，作为调用API服务的客户端。
   * 详细说明如何创建应用并配置其基本信息。
2. **配置访问授权**：
   * 配置应用的访问授权，确保应用具备调用API服务的权限。
   * 提供访问授权配置步骤和注意事项。
3. **申请订阅**：
   * 浏览平台上可用的API服务，选择适合您的API进行订阅，并提交订阅申请。
   * 说明如何搜索API、查看API文档以及申请订阅。
4. **调用API**：
   * 将订阅的API集成到您的系统或应用中，进行调用和使用。
   * 提供集成API的详细指南和示例代码，帮助您快速实现API调用。
